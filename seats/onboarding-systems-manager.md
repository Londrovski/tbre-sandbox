# Onboarding & Systems Manager

Keeps the team's knowledge, tooling and access healthy and well-organised so members and AI tools can find what they need.

## Responsibilities
- **Onboarding** — GitLab and Teams onboarding for new members; Vault guidance.
- **Tooling** — Maintain suggested tooling for all members (VS Code, Lazygit, Everything) and set up Teams filestream.
- **Knowledge hygiene** — Maintain good hygiene of organisation and knowledge distribution; manage repositories.
- **Start_Here index** — Maintain a Start_Here file pointing to all knowledge bases and components.
- **Subteam info packs** — Define info/link/contact packs for the software each subteam needs.

## Key interfaces
- New members
- Vault Access controller
- Web Manager

## Goals
- Team context files in the right place to feed an AI tool / member-facing knowledge network
- A Start_Here file that routes users to every knowledge base
- Defined info packs per subteam
