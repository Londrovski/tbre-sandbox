# Onboarding & Systems Manager

Keeps the team's knowledge, tooling and access healthy and well-organised so members and AI tools can find what they need.

## Responsibilities
- **Onboarding** — GitLab and Teams onboarding for new members; Vault guidance.
  - context: Vault Access controller
  - owns: Onboarding process
  - delivers: New members set up with access & tools
- **Tooling** — Maintain suggested tooling for all members and set up Teams filestream.
  - owns: Tooling list (VS Code, Lazygit, Everything)
  - delivers: Teams filestream set up
- **Knowledge hygiene** — Maintain good hygiene of organisation and knowledge distribution; manage repositories.
  - owns: GitLab repo structure & hygiene
  - delivers: A tidy, well-organised knowledge base
- **Start_Here index** — Maintain a Start_Here file pointing to all knowledge bases and components.
  - owns: The Start_Here file
  - delivers: A Start_Here that routes to everything
- **Subteam info packs** — Define info/link/contact packs for the software each subteam needs.
  - delivers: Info packs per subteam

## Key interfaces
- New members
- Vault Access controller
- Web Manager

## Requirements
- Organised; likes documentation and tidy systems
- Comfortable with GitLab and Teams
- ~2-3 hrs/week
